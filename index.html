Base Styles

Base styles are generally styles with tag selectors. 

button,
input,
optgroup,
select,
textarea {
  color: inherit; 
  font: inherit; 
  margin: 0; 
}

button {
  overflow: visible;
}

Base Styles

Base styles are usually used to reset or normalize.

  • Normalize.css
  • Reset.css

Learn more with given links.

Base Styles vs. Utility Classes

Base styles and utility classes are generally global classes.

Examples of utility classes

.pull-left {
  float: left;
}

.pull-right {
  float: right;
}

Avoiding utility classes is a personal/team choice. I personally prefer using CSS preprocessors mixins rather than creating global utility classes.

Base Styles

Base styles should be carefully regulated and reviewed by team. Reasons are

  • Performance penalty
  • Unused code
  • Unexpected regression (on current projects)

Different projects, teams or scenarios may lead to different usages of base styles.

Naming Conventions

Importance Of Naming Conventions

Ask Java guys.

Naming Layout Styles

Add prefix layout-

.layout-col-1 { }
.layout-col-3 { }
.layout-container { }

Choosing a prefix is your personal/team choice. You can even choose not to have any prefix for layout styles. But you should always have clear documentation on that.

Naming Modules

Modules may have following things to be named

  • itself
  • variations
  • sub-components (not sub-modules)
  • states
.btn { }  /* module itself */
.btn-lg { } /* variations */
.btn-text { } /* sub-components */
.btn-pressed, .btn:hover { } /* states */

Naming Modules - Descendant Selectors?

.btn .btn-large {}
.btn .btn-small {}

Personally, I argue that you should avoid descendant selectors in most cases.

  • Descendant selectors are incrediblely expensive
  • Detach CSS from HTML
  • Modules should do one thing well, so they are generally small

Again, this is a personal/team choice, but should be keep in mind. For large scale projects like Taobao.com, there should be another solution for that. (Thoughts?)

State-based Design

State-based Design

States can be triggered by

  • Classes (.btn-activated, .btn-pressed)
  • Pseudo-classes (.btn:hover)
  • Attribute selectors (http://jsbin.com/tajeyo/1/)
  • Media queries (http://jsbin.com/jimapo/1/)
  • Element queries (in the future)

Code Organization

Group your code into several sections, instead of relying on naming conventions.

  • Complex naming conventions increases visual noise
  • Complex naming conventions is hard to keep consistency (w/o tools)

LESS

Less is a CSS pre-processor, meaning that it extends the CSS language, adding features that allow variables, mixins, functions and many other techniques that allow you to make CSS that is more maintainable, themable and extendable.
